<div class="vespid-section">
  <div class="vespid-meta">What the runtime proves</div>
  <h2>Keep discovery, authorization, approval, and audit in the same story</h2>
  <p>
    vespid is not trying to be an abstract agent manifesto. The point of the current slice is to keep the service contract,
    gateway semantics, and evidence model aligned enough that a generic agent can use ordinary HTTP without bypassing the real trust boundary.
  </p>
  <div class="vespid-project-grid">
    <div class="vespid-project-card">
      <div class="vespid-project-topline">
        <div class="vespid-meta">Contract</div>
        <span class="vespid-status-pill vespid-status-pill--green">Core</span>
      </div>
      <h3>Discoverable service surfaces</h3>
      <p>A service publishes a machine-readable contract through <code>/.well-known/agent-service.json</code> plus OpenAPI agent extensions.</p>
    </div>
    <div class="vespid-project-card">
      <div class="vespid-project-topline">
        <div class="vespid-meta">Authorization</div>
        <span class="vespid-status-pill vespid-status-pill--blue">Core</span>
      </div>
      <h3>Scoped temporary grants</h3>
      <p>Consequential actions stay blocked until the gateway sees a usable grant that matches the task binding and capability.</p>
    </div>
    <div class="vespid-project-card">
      <div class="vespid-project-topline">
        <div class="vespid-meta">Approval</div>
        <span class="vespid-status-pill vespid-status-pill--purple">Core</span>
      </div>
      <h3>Task resume instead of duplicate execution</h3>
      <p>The original task is resumed after grant issuance and approval, which is the key invariant behind the current HTTP demo.</p>
    </div>
    <div class="vespid-project-card">
      <div class="vespid-project-topline">
        <div class="vespid-meta">Evidence</div>
        <span class="vespid-status-pill vespid-status-pill--amber">Core</span>
      </div>
      <h3>Artifacts and audit logs</h3>
      <p>Successful runs return artifacts and append audit evidence so the result is inspectable after the action, not only during it.</p>
    </div>
  </div>
</div>

# Project Atlas

The site is structured around a simple rule: every project should have a crisp public-facing overview before it gets a deep documentation tree.

## Current lanes

- `vespid_voice` — orchestration, voice control, and agent-facing UX.
- `vespid.ai site` — public shell, docs spine, and brand system.
- Future projects can each grow from a single overview page into their own doc subtree.

## What each project page should answer

1. What problem is this trying to solve?
2. Who is it for?
3. What is already working?
4. What remains risky or unresolved?
5. Where do the deeper docs live?

`geo-skill` is the public core of vespid.ai: an open-source GEO skill pack and Python CLI for improving how products, docs, and repositories get discovered, extracted, and cited by AI-native search systems.

## Project overview

A lot of teams already know AI search matters, but the work still lands as vague advice:

- write clearer copy
- add some schema
- publish an `llms.txt`
- hope ChatGPT Search cites the right page

`geo-skill` exists to turn that into operational work instead:

- reusable GEO skills for agents and operators
- a CLI for audit, generation, benchmark inspection, and report comparison
- explicit support for machine-readable public artifacts
- a repeatable workflow for improving public discoverability without hand-wavy SEO language

## What is already real

The public repository already includes:

- Hermes / Claude Code / Codex-ready GEO skills
- a Python CLI for audits, generators, benchmark inspection, and report comparison
- support for `llms.txt`, `robots.txt`, schema generation, and page-outline workflows
- practical coverage for docs sites, product pages, changelogs, pricing pages, trust pages, comparison pages, and OSS discoverability
- tagged public releases that make the workflow versioned and reusable

That means `geo-skill` is already a runnable toolchain, not a GEO concept deck.

## Trust boundary

The important boundary is between GEO work that is explicit and inspectable versus GEO work that turns into manipulative folklore.

`geo-skill` is meant to improve:

- factual clarity
- machine-readable structure
- retrieval paths
- documentation legibility
- durable public artifacts that AI systems can cite reliably

It is not trying to game platforms with hidden prompts or fake signals.

## Why it matters

AI-native search systems do not just reward brand size. They reward public surfaces that are easy to read, easy to extract, and easy to trust.

That means a technically strong product can still lose visibility if:

- the homepage is vague
- the docs do not expose stable facts
- the right URLs are not reinforced with machine-readable artifacts
- repository and website surfaces tell different stories

`geo-skill` exists to close that gap with repeatable tooling.
